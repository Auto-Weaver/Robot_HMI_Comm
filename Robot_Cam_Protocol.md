# 指纹模组AOI项目 机械臂与PC算法端通信协议

## 1. 通信概述

| 项目         | 内容                                              |
| :----------- | :------------------------------------------------ |
| 通信双方     | 机械臂客户端（Client） ↔ PC算法端服务器（Server） |
| 连接方式     | TCP Socket                                        |
| 连接发起方   | 机械臂主动连接 PC算法端                           |
| PC算法端角色 | 服务器（Server）                                  |
| PC算法端IP   | `192.168.57.10`（暂定）                           |
| PC算法端端口 | `30000`                                           |
| 消息格式     | JSON                                              |
| 编码格式     | UTF-8                                             |

说明：机械臂连接成功或失败，由机械臂自身函数判断，不作为协议交互内容。

## 2. JSON格式约定

| 方向          | 格式                                                         | 说明                                                         |
| :------------ | :----------------------------------------------------------- | :----------------------------------------------------------- |
| 机械臂 → 算法 | `{"arm":"<command>"}`                                        | 机械臂所有请求统一使用 `arm` 字段                            |
| 算法 → 机械臂 | `{"distance":<number>}` / `{"quality":"<value>"}` / `{"result":"<value>"}` | 算法返回字段根据返回内容区分：测距使用 `distance`，图像质量使用 `quality`，检测结果使用 `result` |

机械臂请求示例：`{"arm":"get_distance"}`
算法返回示例：`{"distance":125.6}`、`{"quality":"pass"}`、`{"result":"OK"}`

## 3. 字段定义与数据类型

| 字段名     | 方向          | 数据类型 | 取值示例           | 说明                     |
| :--------- | :------------ | :------- | :----------------- | :----------------------- |
| `arm`      | 机械臂 → 算法 | string   | `"get_distance"`   | 机械臂发送的请求命令字段 |
| `distance` | 算法 → 机械臂 | number   | `125.6`            | 算法返回的测距值字段     |
| `quality`  | 算法 → 机械臂 | string   | `"pass"`、`"blur"` | 算法返回的图像质量字段   |
| `result`   | 算法 → 机械臂 | string   | `"OK"`、`"NG"`     | 算法返回的检测结果字段   |

## 4. 机械臂发送命令定义

| JSON示例                     | 命令值             | 含义                                                       | 算法端是否需要返回 |
| :--------------------------- | :----------------- | :--------------------------------------------------------- | :----------------- |
| `{"arm":"prepare_distance"}` | `prepare_distance` | 准备测距。通知算法端提前准备获取超声测距传感器数据         | 否                 |
| `{"arm":"get_distance"}`     | `get_distance`     | 当前测距值。机械臂到达上料处后，请求算法返回当前超声测距值 | 是                 |
| `{"arm":"top_arrived"}`      | `top_arrived`      | 上表面到位。机械臂到达当前产品上表面拍照位置               | 是                 |
| `{"arm":"retake_top"}`       | `retake_top`       | 上表面重拍。机械臂调整位置后，对当前产品上表面进行重新拍照 | 是                 |
| `{"arm":"down_arrived"}`     | `down_arrived`     | 下表面到位。机械臂到达当前产品下表面拍照位置               | 是                 |
| `{"arm":"retake_down"}`      | `retake_down`      | 下表面重拍。机械臂调整位置后，对当前产品下表面进行重新拍照 | 是                 |
| `{"arm":"get_result"}`       | `get_result`       | 请求结果。下表面图像质量通过后，机械臂请求当前产品检测结果 | 是                 |

## 5. 算法端返回定义

### 5.1 测距返回

| 对应请求       | JSON示例             | 数据类型 | 说明                                                 |
| :------------- | :------------------- | :------- | :--------------------------------------------------- |
| `get_distance` | `{"distance":125.6}` | number   | 返回当前超声测距传感器的测距值，供机械臂计算吸取高度 |

### 5.2 图像质量返回

| 对应请求                       | 返回值               | 数据类型 | 含义                                         |
| :----------------------------- | :------------------- | :------- | :------------------------------------------- |
| `top_arrived` / `retake_top`   | `{"quality":"pass"}` | string   | 上表面图像质量通过，机械臂可前往下一个位置   |
| `top_arrived` / `retake_top`   | `{"quality":"blur"}` | string   | 上表面图像离焦，机械臂需调整位置后重拍       |
| `down_arrived` / `retake_down` | `{"quality":"pass"}` | string   | 下表面图像质量通过，机械臂可继续请求检测结果 |
| `down_arrived` / `retake_down` | `{"quality":"blur"}` | string   | 下表面图像离焦，机械臂需调整位置后重拍       |

规则：上表面和下表面均最多允许2次位置调整。第2次重拍后，算法端必须返回 `{"quality":"pass"}`，机械臂继续后续流程。

### 5.3 检测结果返回

| 对应请求     | 返回值            | 数据类型 | 含义               |
| :----------- | :---------------- | :------- | :----------------- |
| `get_result` | `{"result":"OK"}` | string   | 当前产品检测合格   |
| `get_result` | `{"result":"NG"}` | string   | 当前产品检测不合格 |

## 6. 通信流程

1. 设备上电。 
2. 机械臂作为客户端主动连接PC算法端服务器（IP：192.168.57.10，端口：30000）。 
3. 设备启动按钮触发启动信号，机械臂自动程序开始运行。 
4. 机械臂前往上料处前，发送 `{"arm":"prepare_distance"}`，仅通知算法准备测距，不要求返回。 
5. 机械臂到达上料处后，发送 `{"arm":"get_distance"}`，算法返回当前测距值，例如 `{"distance":125.6}`。 
6. 机械臂根据测距值完成料盘吸取。 
7. 机械臂移动到检测点，开始遍历料盘中的产品上表面。 
8. 每到一个上表面位置，机械臂发送 `{"arm":"top_arrived"}`。 
9. 算法返回： - `{"quality":"pass"}`：机械臂前往下一个位置。 - `{"quality":"blur"}`：机械臂调整位置后发送 `{"arm":"retake_top"}`。 
10. 若上表面连续离焦，机械臂最多调整2次；第2次重拍后，算法必须返回 `{"quality":"pass"}`。 
11. 上表面全部遍历完成后，机械臂开始遍历产品下表面。 
12. 每到一个下表面位置，机械臂发送 `{"arm":"down_arrived"}`。 
13. 算法返回： - `{"quality":"pass"}`：机械臂继续请求当前产品检测结果。 - `{"quality":"blur"}`：机械臂调整位置后发送 `{"arm":"retake_down"}`。 
14. 若下表面连续离焦，机械臂最多调整2次；第2次重拍后，算法必须返回 `{"quality":"pass"}`。 
15. 当下表面图像质量通过后，机械臂发送 `{"arm":"get_result"}`。 
16. 算法返回： - `{"result":"OK"}` - `{"result":"NG"}` 
17. 机械臂获取结果后，前往下一个产品位置，重复上述流程。 
18. 全部产品上下表面检测完成后，本轮检测结束。

## 7. 命令与返回值汇总

| 机械臂发送                   | 算法返回                                    | 说明                 |
| :--------------------------- | :------------------------------------------ | :------------------- |
| `{"arm":"prepare_distance"}` | 无                                          | 准备测距             |
| `{"arm":"get_distance"}`     | `{"distance":number}`                       | 获取当前测距值       |
| `{"arm":"top_arrived"}`      | `{"quality":"pass"}` / `{"quality":"blur"}` | 上表面到位           |
| `{"arm":"retake_top"}`       | `{"quality":"pass"}` / `{"quality":"blur"}` | 上表面重拍           |
| `{"arm":"down_arrived"}`     | `{"quality":"pass"}` / `{"quality":"blur"}` | 下表面到位           |
| `{"arm":"retake_down"}`      | `{"quality":"pass"}` / `{"quality":"blur"}` | 下表面重拍           |
| `{"arm":"get_result"}`       | `{"result":"OK"}` / `{"result":"NG"}`       | 获取当前产品检测结果 |

版本说明：本版仅按当前已确认流程定义基础通信协议，后续如增加产品ID、序号、超时机制、异常码、心跳机制，可在此基础上扩展。
